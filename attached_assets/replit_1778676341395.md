# KAPPA Heartbeat Tracker

## Overview

Standalone device heartbeat and status tracking service. Devices (PCs, phones, sensors, IoT) register themselves and send periodic heartbeats. The service tracks online/offline status, stores device metadata, and exposes REST + WebSocket APIs.

## Stack

- **Monorepo tool**: pnpm workspaces
- **Node.js version**: 24
- **Package manager**: pnpm
- **TypeScript version**: 5.9
- **API framework**: Express 5
- **Database**: PostgreSQL + Drizzle ORM
- **Validation**: Zod (`zod/v4`), `drizzle-zod`
- **API codegen**: Orval (from OpenAPI spec)
- **Build**: esbuild (CJS bundle)
- **Frontend**: React 18 + Vite + Tailwind CSS + shadcn/ui
- **WebSocket**: `ws` library
- **Charts**: Recharts

## Database Tables

- **devices** — registered devices with online status, capabilities, metadata
- **heartbeats** — timestamped heartbeat records with system metrics (24h retention)
- **alerts** — timeout/reconnect/anomaly alerts with severity levels
- **sensor_readings** — sensor data from devices (6h retention)

## API Endpoints

### Devices
- `POST /api/devices/register` — register/update device
- `GET /api/devices` — list all (filterable by type/online)
- `GET /api/devices/stats` — totals, online/offline, by-type breakdown
- `GET /api/devices/:deviceId` — full detail with heartbeats & alerts
- `DELETE /api/devices/:deviceId` — remove device

### Heartbeat
- `POST /api/heartbeat` — send heartbeat with metadata
- `GET /api/heartbeat/:deviceId/history` — heartbeat history

### Sensors
- `POST /api/sensors` — ingest sensor readings
- `GET /api/sensors/:deviceId` — readings by type/timeframe
- `GET /api/sensors/:deviceId/latest` — latest per sensor type

### Alerts
- `GET /api/alerts` — list (filterable)
- `GET /api/alerts/active` — unacknowledged only
- `POST /api/alerts/:id/acknowledge` — acknowledge alert

### KAPPA Integration
- `GET /api/kappa/status` — full system summary
- `POST /api/kappa/command` — send command to device via WebSocket

### Agent Scripts
- `GET /api/agents/pc` — Python script for Windows/Linux/Mac
- `GET /api/agents/phone` — Python script for Termux (Android)
- `GET /api/agents/bash` — Minimal bash heartbeat script

## WebSocket

- Endpoint: `/ws?deviceId=<id>`
- Messages: register, heartbeat, sensors, alert, command
- Heartbeat every 15s, timeout at 45s

## Background Jobs

- **Heartbeat Monitor** (every 15s) — marks devices offline after 45s timeout
- **Cleanup** (every hour) — removes old heartbeats (24h), sensors (6h), ack'd alerts (7d)

## Key Commands

```bash
pnpm --filter @workspace/api-spec run codegen    # Regenerate API types
pnpm --filter @workspace/db run push             # Push schema to DB
pnpm --filter @workspace/api-server run dev      # Start API server
pnpm --filter @workspace/kappa-monitor run dev   # Start frontend
```

## TypeScript & Composite Projects

Every package extends `tsconfig.base.json` which sets `composite: true`. The root `tsconfig.json` lists all packages as project references.

- **Always typecheck from the root** — run `pnpm run typecheck`
- **`emitDeclarationOnly`** — only emit `.d.ts` during typecheck; bundling via esbuild/vite

## Packages

### `artifacts/api-server` (`@workspace/api-server`)
Express 5 + WebSocket server. Routes in `src/routes/`, WebSocket in `src/lib/websocket.ts`, background jobs in `src/lib/background.ts`.

### `artifacts/kappa-monitor` (`@workspace/kappa-monitor`)
React dashboard at `/`. Dark mode by default. Shows device grid, alert feed, stats bar. Device detail with heartbeat charts, sensor readings, command buttons, agent downloads.

### `lib/db` (`@workspace/db`)
Drizzle ORM schemas: devices, heartbeats, alerts, sensor_readings.

### `lib/api-spec` (`@workspace/api-spec`)
OpenAPI 3.1 spec. Run codegen: `pnpm --filter @workspace/api-spec run codegen`
