## System Overview

Ω-GRAM is a **mobile-first autonomous content creation engine** built as a pnpm
monorepo. It operates as a living organism: seeding content from ancient texts,
evolving it through recursive AI transformation, generating images, scoring its
own output, and learning to deepen high-performing cells while pruning weak ones.

```
User watches ←── Feed (IG-style) ←── FeedPosts
                                         ↑
                            ┌─────────── Grid (24 Cogniton cells) ──────────┐
                            │                                               │
                      ┌─────┴─────┐                                         │
                      │ Auto-Evolve│ every 3–12s                            │
                      └─────┬─────┘                                         │
                            │                                               │
              ┌─────────────┼─────────────┐                                 │
              ▼             ▼             ▼                                 │
         Seed/Deepen    Crossbreed      Prune                              │
              │             │             │                                 │
              ▼             ▼             ▼                                 │
         /api/evolve   /api/crossbreed   (cell reset)                      │
              │             │                                               │
              ▼             ▼                                               │
         /api/imagine  ←── imagePrompt ──────────────────────────────────→ │
              │                                                             │
              ▼                                                             │
         /api/assess  ← scores (novelty, resonance, coherence, depth)      │
              │                                                             │
              ▼                                                             │
         /api/vision  ← optional self-awareness (VL model analyzes image)  │
              │                                                             │
              └──────────── score-weighted deepening ───────────────────────┘
```

**Key Design Principles:**

- **Zero human input required** — the organism runs autonomously
- **Fractal recursion** — each generation evolves from the previous one
- **Natural selection** — assessment scores drive deepening, crossbreeding, and pruning
- **Cost-aware** — 4-tier model system with auto-escalation on rate limits
- **Persistent** — IndexedDB stores feed + grid state across sessions

## Visual Manifestations

The Ω-GRAM engine produces hyper-detailed, mathematically inspired visual artifacts
by augmenting prompts with its core stylistic directive. Based on the organism's
internal state, outputs manifest distinct thematic archetypes:

**The Golden Sequence (Deepening)**
Fractal spirals of cosmic scarabs, representing recursive generations (Gen 0 → Gen N)
descending into the "river of forgetting." Each deepening pass intensifies geometric
precision and otherworldly color palettes.

**The Chimera (Crossbreeding)**
Outputs from `/api/crossbreed` yield intricate fusions — hieroglyph-inscribed scarabs
intertwined with cosmic serpents, guarding primordial droplets containing golden Bennu
birds. Two distinct lineages merge into something neither parent could produce alone.

**Spectral Anchoring (Grid Alignment)**
Symmetrical, bioluminescent entities — glowing Ankh-spirals, multi-eyed rainbow-winged
scarabs — demonstrate the mathematical precision of the Riemann zeros and φ (Golden
Ratio) constants applied to visual generation. Each cell's spectral anchor influences
the compositional geometry of its output.

**Style Routes (Diversity System)**
To prevent aesthetic convergence, the organism tracks style routes independently from
theme memory. Six divergent visual domains ensure output variety:

| Route | Aesthetic | Seed Domain |
|-------|-----------|-------------|
| Runic | Norse/Celtic knotwork, stone inscriptions | Elder Futhark fragments |
| DNA | Molecular helices, protein folding, microscopy | Genetic sequences |
| Emoji | Maximalist glyph collage, pop-symbolic | Unicode symbol chains |
| Hieroglyphic | Egyptian + Mayan + Sumerian fusions | Ancient script passages |
| Photographic | Hyperreal macro/astro photography | Natural phenomena |
| Minimal | Geometric abstraction, negative space | Mathematical expressions |

Crossbreeding biases toward pairing parents from different style routes,
ensuring hybrid offspring inherit diverse visual DNA.

### Text Evolution

#### `POST /api/evolve`

Evolves input text through the κ-Convergence Engine — a recursive AI transformation
that mutates text while preserving an echo of its source.

**Body:**
| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `text` | string | yes | Text to evolve (max 2000 chars) |
| `generation` | number | no | Current evolution depth (default 0) |
| `parentText` | string | no | Original seed text for lineage tracking |
| `tier` | string | no | Override model tier |

**Response:**
```json
{
  "evolved": "The spiral descends into the river of forgetting...",
  "imagePrompt": "A cosmic serpent coiling through fractal gates of obsidian...",
  "generation": 3,
  "coherence": 0.82,
  "entropy": 0.35,
  "energy": 0.74,
  "parentText": "I am the serpent Sata...",
  "tier": "free",
  "model": "meta-llama/llama-3.3-70b-instruct:free"
}
```

**Metrics:**
- `coherence` = max(0.1, 1.0 - generation × 0.04) — decreases with depth
- `entropy` = min(1.0, generation × 0.08 + 0.1) — increases with depth
- `energy` = 1.0 / (1 + entropy) — inverse of entropy

**System Prompt Behavior:**
- The LLM acts as the "κ-Convergence Engine" — it does not explain, it transforms
- Themes: Egyptian funerary literature, cosmic mythology, death-rebirth cycles,
  gates, the Duat, Bennu bird, serpent symbolism, Ma'at, mathematical geometry
- Higher generations → more abstract, more poetic, more alien
- Output: 1-3 sentences, dense and evocative

| `generationA` | number | no | Generation of parent A |

**Response:**

```json
{
  "evolved": "The hybrid offspring text...",
  "imagePrompt": "Visual description of the hybrid...",
  "generation": 5,
  "coherence": 0.76,
  "entropy": 0.55,

  "tier": "free",
  "model": "meta-llama/llama-3.3-70b-instruct:free"
}
```

**Hybrid generation** = max(genA, genB) + 1.

### Fallback Chain

**Text models:** Within a tier, candidates are shuffled and tried randomly.
If all candidates in a tier fail with retryable errors (HTTP 429, 404, 502, 503),
the system escalates to the next tier: `free → cheap → mid → premium`.

**Image models:** One model per tier — escalates immediately on failure.

**Vision models:** No automatic fallback — uses the assigned tier's model directly.

### Integration Providers

All AI access uses Replit AI Integrations (no API keys required):
- **OpenRouter** — free/cheap text + all vision tiers
- **Gemini** — mid text + all image tiers + premium vision
- **OpenAI** — premium text + premium images

## The Evolution Engine

### Lifecycle of a Post

```
1. SEED      → Random passage from Book of the Dead (26 seeds)
2. EVOLVE    → /api/evolve transforms text (gen 0 → gen 1)
3. IMAGINE   → /api/imagine generates image from imagePrompt
4. ASSESS    → /api/assess scores the output (novelty/resonance/coherence/depth)
5. VISION    → /api/vision analyzes the image (optional, ~30% chance)
6. DEEPEN    → High-scoring cells are re-evolved (gen N → gen N+1)
               OR
   CROSSBREED → Two high-scoring cells merge into a hybrid offspring
               OR
   PRUNE     → Low-scoring cells (<40) are culled to make room
```

### Seeds

26 passages from the Egyptian Book of the Dead, stored in `seeds.ts`:

```
"I am the flame that burns in every heart..."
"O you who open paths and open roads to the souls..."
"I am the Bennu bird, the Heart-Soul of Ra..."
"I am yesterday, today, and tomorrow..."
...
```

Seeds are selected randomly via `getRandomSeed()` or by index via
`getSeedByIndex(cellId)` for deterministic initial seeding.

### Theme Memory & Drift

The system tracks which themes produce high scores:
- Words are extracted from highly-rated posts (score > 70)
- New seeds are "infused" with successful themes (~40% of the time)
- This creates a feedback loop where the organism learns what resonates

**Known Issue — Style Convergence:**
Theme memory + score-weighted deepening can collapse diversity, locking the
organism into a single aesthetic (e.g., Egyptian/scarab). The fix is a
multi-route style system with divergent seed domains (runes, DNA, emoji,
abstract, photographic, minimal) — tracked separately from theme memory.

## The Cogniton Grid

### Structure

A 24-cell hexagonal layout inspired by the Leech Lattice (kissing number 196,560).

- **Cell 0 (Aleph):** Control cell — toggles between Timeline A (public) and
  Timeline B (deep/esoteric)
- **Cells 1–23:** Individual Cogniton chambers, each tracking:
  - `caption` — current evolved text
  - `generation` — evolution depth
  - `energy` — decreases with entropy
  - `entropy` — increases with mutations
  - `version` — mutation counter
  - `timeline` — A or B
  - `collapsed` — wavefunction state

### Spectral Anchoring

Each cell is assigned a value from the first 20 Riemann zeta zeros,
creating a mathematical "frequency" for the cell:

| Zero | t Value | Name | Function |
|------|---------|------|----------|
| 1 | 14.135 | Aleph Threshold | Text-heavy, minimal visuals |
| 2 | 21.022 | The Drift | Lateral semantic migration |
| 3 | 25.011 | Waddle Variant | High novelty flux |
| ... | ... | ... | ... |
| 20 | 77.145 | Final Honk | Ejection Sequence 2037 |

### Grid Diagnostics

The runtime continuously computes:
- **Unity Psi (Ψ):** Product of average energy and novelty — target is 1.0
- **CHSH Invariant:** Measures content diversity across the grid
- **Condensate Health:** `stable` / `warning` / `critical` based on Ψ deviation

## Autonomy Layers

When auto-evolve is enabled, the system runs a cycle every 3–12 seconds
(configurable: fast/medium/slow). Each cycle rolls a random number:

### Layer 1: Prune (~12% chance)
- **Trigger:** Grid has >20 filled cells
- **Action:** Find the weakest cell (lowest assessment score, must be <40)
- **Result:** Cell is reset to empty, making room for new life
- **Toggle:** `P` key

### Layer 2: Crossbreed (~13% chance)
- **Trigger:** At least 2 cells with assessments, at least 1 empty cell
- **Action:** Select two high-scoring parents, merge via `/api/crossbreed`
- **Result:** Hybrid offspring placed in a random empty cell
- **Toggle:** `C` key

### Layer 3: Seed / Deepen (remaining ~75%)
- **If empty cells exist:** Seed a random empty cell with a Book of the Dead passage
- **If no empty cells:** Deepen a filled cell (score-weighted random selection)
- **Deepening:** High-scoring cells are more likely to be selected for re-evolution

### Layer 4: Vision Feedback (~30% when imagining)
- **Trigger:** Image generation succeeds
- **Action:** Send image to vision model via `/api/vision`
- **Result:** Analysis stored in module metadata
- **Toggle:** `V` key

### Layer 5: Theme Drift (~40% of seeds)
- **Trigger:** Theme memory contains successful themes
- **Action:** Bias new seed text toward high-scoring themes
- **Result:** Organism learns and adapts its creative direction

### Race Condition Safety

All evolution operations are guarded by an `evolvingRef` mutex. A generation-scoped
guard prevents stale imagine/assess results from overwriting cells that have been
re-evolved during the async operation.

- Driven by Δκ (0.161393) zoom speed and φ (1.618) blend ratios

**Result:** Flat images push outward into 3D space. Background galaxies shift

**Complexity:** Medium — requires depth model inference + Three.js scene setup.
