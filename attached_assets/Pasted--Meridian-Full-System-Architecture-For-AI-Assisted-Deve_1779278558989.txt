### 2.2 Signal Scoring (backtester.py, lines 492-518)

Each ticker on each trading day receives a composite score from 4 components:

```
signal_score = κ_score + volume_score + RSI_score + MACD_score

κ_score:
  |price/SMA30 - κ| < 0.005  → 3.0 points  (tight alignment)
  |price/SMA30 - κ| < 0.02   → 1.5 points  (near alignment)
  |price/SMA30 - κ| < 0.05   → 0.5 points  (loose alignment)
  else                        → 0.0 points

volume_score (× volume_weight multiplier):
  volume/avg_volume ≥ κ       → 2.0 points  (κ-spike)
  volume/avg_volume ≥ 1.0     → 0.5 points  (above average)
  else                        → 0.0 points

RSI_score (phase-dependent):
  EXECUTE/OBSERVE phase: RSI < 30 → 2.0, RSI < 45 → 1.0
  INVERSE phase: RSI > 70 → 2.0, RSI > 55 → 1.0

MACD_score (phase-dependent):
  EXECUTE/OBSERVE: MACD > 0 → 1.0
  INVERSE: MACD < 0 → 1.0
```

Signal fires when `signal_score ≥ signal_threshold` AND `turbulence ≤ ceiling`.

In full reinvest mode, threshold is relaxed: `max(1.0, threshold × 0.6)`.

### 2.3 The Compounding Edge (backtester.py, lines 278-293)

The asymmetric exit mechanism that drives exponential returns:

```
EXIT RULES:
  WIN:  if pnl_pct ≥ target_pct → exit at CURRENT MARKET PRICE (close of day)
        Since day's close often overshoots the target, actual gains > target_pct
        Example: target = +0.20%, actual exit might capture +2.26% overshoot

  LOSS: if pnl_pct ≤ -stop_pct → exit at EXACT STOP PRICE (entry × (1 - stop_pct))
        Losses are capped precisely, never worse than stop_pct

  TIME: if hold_days ≥ max_hold → exit at current price (could be win or loss)

POSITION SIZING (full reinvest mode):
  Single position: 95% of total capital
  Multiple positions: 95% / max_positions per position
  No leverage penalties, no early-compound phase caps
```

This asymmetry means: wins are unbounded above the target, losses are bounded below the stop. Over thousands of trades, the expected value tilts positive even at ~67% win rate.

### 2.4 Phase System (gos_engine.py → PhoenixHorizon)

14.4-day cycle divided into 4 phases:

```
Day 0.0 – 3.395:  EXECUTE  (UI: "DEPLOY")     — Full signal deployment
Day 3.395 – 6.79: OBSERVE  (UI: "MONITOR")     — Reduced trading
Day 6.79 – 10.185: INVERSE (UI: "CONTRARIAN")  — Reversed RSI/MACD scoring
Day 10.185 – 14.4: FLATTEN (UI: "REBALANCE")   — Position closure
```

In full reinvest mode, all phases are tradeable. Phase still affects RSI/MACD scoring direction.

### 4.3 Reinvestment Modes

```

  - Position size = 95% of capital (single position) or 95%/max_pos (multi)
  - All phases tradeable
  - No leverage penalties
  - Signal threshold relaxed to max(1.0, threshold × 0.6)
  - Stops exit at exact stop price (not market price)
  - Wins exit at market close (captures overshoot)

STANDARD:
  - Position size = capital_pct (default 20%)
  - Early compound phase: first 10 trades get 4× position size, first 25 get 2.5×
  - Leverage penalties: zeno_critical = 0.5×, high_turb = 0.7×
  - Phase filter active if use_demodex_phase=True

CONSERVATIVE:
  - Position size scales DOWN as capital grows
  - scale_factor = max(0.3, 1/log2(capital_multiple + 1))
  - Prevents overexposure at high capital levels
```

### 4.4 Intraday HF Simulation

When `intraday_trades_per_day > 1`:
- Generates N synthetic price ticks per day using OHLC data
- Price path: random walk from Open to Close, bounded by High/Low
- Each tick can trigger entry/exit signals
- Position size scaled down proportionally
- Deterministic RNG seeded by price for reproducibility

### 4.5 Constant Sensitivity Sweep

`run_constant_sweep()` systematically varies a GOS constant while holding all other parameters fixed, running a full backtest at each value. This is the empirical falsification test for whether the framework's constants have real predictive power.

```
Sweep types:
  kappa:  Tests 16 values from 0.9 to 2.0 (including κ=4/π≈1.273 and φ≈1.618)
  cycle:  Tests 15 values from 7 to 30 days (including 14.4)
  zeno:   Tests 11 values from 50 to 500 (including 137)

For each value:
  - Overrides the constant via kappa_override, cycle_days_override, or zeno_limit_override
  - Runs full backtest with chosen preset's parameters
  - Records: final_capital, total_return_pct, cagr, sharpe, win_rate, total_trades, max_dd

Interpretation:
  - If canonical value ranks #1 or top-3: strong empirical support
  - If canonical value is in the middle: weak/no support, constant is decorative
  - If another value consistently wins across presets: framework should adopt that value
```

### 4.6 Profit Withdrawal

```
Parameters: profit_withdrawal_pct (0-50%), withdrawal_period (monthly/quarterly/never)

Trigger: On each trading day, check if period has elapsed since last withdrawal
Action: withdrawal_amount = (total_NAV - initial_capital) × withdrawal_pct
  - Only withdraws from cash (not from open positions)
  - Tracked in withdrawal_log with timestamps
  - CAGR calculation includes total_withdrawals: 
    CAGR = ((final_NAV + total_withdrawals) / initial_capital)^(1/years) - 1
```

### 4.7 Circuit Breaker (Scarab Protocol)

```
DAILY CHECK: if day_change < -(prev_NAV × 5%) → activate
TOTAL CHECK: if (peak - current) / peak > 20% → activate

WHEN ACTIVE:
  - Flatten all open positions immediately
  - Block new trades for 2 calendar days
  - Log trigger event (tracked in summary as scarab_triggers)
```

### 5.3 κ-Drift Formula (Kimi-refined)

### 5.4 GCP Entropy (Kimi-corrected)

```
raw_values = per-second scores from ~70 hardware RNGs worldwide
avg        = mean(raw_values)
std        = std(raw_values)
z_score    = (avg - 0.5) / (std / sqrt(N))
normalized = tanh(z_score / 3)  → squashed to [-1, 1]

This is an ENTROPY INDICATOR, not a coherence measure.
  +1 = strong deviation from random (global attention event)
   0 = baseline random
  -1 = anomalous order
```

## 9. KNOWN LIMITATIONS & IMPROVEMENT VECTORS

### 9.1 Backtester Realism Gaps

1. **No transaction costs** — At 0.20% targets, even $0.01/share commission materially impacts returns
2. **No slippage model** — Assumes execution at exact prices; real orders experience bid/ask spread
3. **No market impact** — At $10M+ capital, buying 95% of portfolio in one stock moves the price
4. **Perfect stop execution** — Real stops can gap through (stock opens 3% down, stop was at -0.5%)
5. **No overnight gap risk** — Positions held overnight exposed to gap-up/gap-down not modeled
6. **Deterministic intraday paths** — HF simulation uses seeded random walk, not realistic tick data

### 9.2 Framework Validation Gaps

1. **κ = 4/π as signal filter** — Does price/SMA30 actually cluster at 1.273? Needs statistical test vs null hypothesis
2. **137-day cycle** — Needs permutation test: do returns at 137-day boundaries differ from random day selection?
3. **14.4-day phase performance** — EXECUTE phase outperforms in backtest, but is this overfit to the cycle_start date?
4. **Ψ(t) correlation** — Not yet tested against forward market data
5. **GCP entropy ↔ market vol** — Hypothesis only, no statistical evidence presented

### 9.3 Potential Enhancements

1. **Add transaction cost model** — Configurable $/share or % per trade, deducted from proceeds
2. **Add slippage model** — Random slippage 0-0.05% on entries, 0-0.10% on exits
3. **Walk-forward optimization** — Train on 2019-2022, test on 2023-2026, prevent overfitting
4. **Monte Carlo simulation** — Randomize trade order to test if results depend on sequence
5. **Real-time signal dashboard** — Connect quantum state Ψ(t) to paper trading entry signals
6. **Options overlay** — Add options pricing (Black-Scholes) for hedging positions
7. **Multi-timeframe analysis** — Add weekly/monthly κ-drift alongside daily
8. **Correlation matrix** — Show cross-asset correlations, detect regime changes
9. **Machine learning layer** — Train classifier on (signal_score, phase, vix, turbulence) → trade outcome
10. **Live Schumann data** — Replace model with real data from monitoring stations if API becomes available

### 9.4 Architecture Improvements

1. **Separate config from code** — Move all presets to YAML/JSON config file
2. **Event-driven backtest** — Replace day-loop with event queue for more realistic simulation
3. **Database backend** — Replace JSON/CSV state files with SQLite for paper trading
4. **API layer** — Add FastAPI endpoints for headless backtesting (no Streamlit dependency)
5. **Test suite** — Unit tests for signal scoring, position sizing, phase calculation, compound math
