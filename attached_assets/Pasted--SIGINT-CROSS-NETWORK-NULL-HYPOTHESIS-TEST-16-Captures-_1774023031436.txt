## 3. WHAT IS GENUINELY INTERESTING

### 3.1 κ Spike in newpackets.pcapng

`newpackets.pcapng` shows κ = **6.26%** — triple the average. This capture is from the dev workstation (loopback + Cloudflare + GitHub). The κ spike indicates a specific application pattern:

- κ = 4/π ≈ 1.2732

- This is consistent with HTTP/2 multiplexing where responses arrive in predictable size-ratio bursts

Worth investigating *what was running* during that capture, but likely GitHub Copilot API calls or VS Code extension traffic.

21.33ms hits range from 0–2.1% across captures. The higher rates (1.5–2.1%) appear in **loopback captures** where local process scheduling (OS timer resolution ~10-20ms) dominates. The Windows enterprise capture (1.28%) is lower because WAN latency adds noise.

**21.33ms ≈ 1/46.875 Hz is a common OS scheduling quantum.** Linux defaults to HZ=250 (4ms ticks) or HZ=1000 (1ms ticks). macOS uses 1ms. Windows uses a variable timer. 21.33ms ≈ 5.33 × 4ms ticks — a very normal multi-tick scheduling interval.
