# DEEP RESEARCH PROMPT: LAYERED, HORIZONTALLY‑SCALING AGENT ARCHITECTURE WITH WEBGPU LOCAL LLMS

## Objective
Design a **12‑dimensional (or n‑layer) wiki‑style architecture** where each layer is a semi‑transparent “canvas” containing horizontally scaled agent clusters. The user interacts through a **chat interface at the bottom layer** (access point). Layers above can be hidden, masked, or partially visible. Some layers have **sub‑layers** (vertical recursion). All inference runs **locally** via WebGPU LLMs (e.g., Gemma‑4) and small, specialized agents (e.g., from Hugging Face `smolagents`).

The architecture must support:
- **Horizontal agent teams** within a single layer (e.g., 3 agents: Designer, Coder/optimizer, Engineer).
- **Hypervisor synthesis layer** that consumes outputs from multiple layers and feeds back into lower layers (like a gradient in Photoshop, but semantic).
- **Layer masks** that control which parts of a layer affect visible layers.
- **Hidden layers** that influence visible layers indirectly (e.g., via the hypervisor).
- **Scalability** – each layer can spawn any number of horizontal agent groups.

## Core Technologies to Investigate & Integrate

### 1. WebGPU Local LLM Inference
- **Gemma‑4 (or any 7B/2B) via WebGPU** – real‑time, no server.
- Model loading, KV caching, streaming responses in browser.
- **Quantization** – 4‑bit or 8‑bit to fit GPU memory.

### 2. Small, Specialized Local Agents
- **`smolagents` (HF)** – lightweight, runs on CPU/GPU.
- **LangChain / LiteLLM** – orchestration, but with local endpoints (Ollama, LM Studio).
- **Web Workers** – each agent in its own thread for parallelism.

### 3. Layered Architecture (Inspired by Photoshop)
- **Layer stack** – bottom to top: Chat UI → Agent orchestration → Hypervisor → Hidden knowledge layers → Raw data layers.
- **Blend modes** – “Add” (sum outputs), “Multiply” (constrain), “Screen” (union of agent suggestions), “Difference” (error propagation).
- **Opacity** – 0.0 to 1.0, controls how strongly a layer influences the layer above.
- **Layer masks** – binary or continuous masks (e.g., an agent only applies to certain token positions or document sections).
- **Hidden layers** – no direct UI, but their output is weighted into the hypervisor.

### 4. Horizontal Scaling Within a Layer
- **Agent team** – multiple agents with distinct roles (design, code, engineering) running in parallel.
- **Consensus mechanism** – majority vote, weighted by confidence, or hypervisor arbitration.
- **Elastic scaling** – spawn/kill agents based on load.

### 5. Hypervisor Synthesis Layer
- Receives outputs from **all lower layers** (including hidden ones) as weighted tensors.
- Produces a **synthesized update** that is fed back into the chat layer (user‑facing) and optionally into hidden layers (as a gradient).
- **Hypervisor itself** is a WebGPU LLM with a special system prompt that understands layer semantics.

## Research Questions to Answer

### Layer Mechanics
1. What data structure represents a layer? (e.g., tensor of shape `[batch, sequence, hidden_dim]` plus mask, opacity, blend mode)
2. How are layers composed? Order‑dependent? Can they be grouped?
3. How to implement a “hidden” layer that influences visible layers without being inspectable? (e.g., differential privacy, encrypted weights, or simply omitted from UI enumeration)

### Agent Integration
1. How to run 3+ agents (Designer, Coder, Engineer) on the same local WebGPU LLM instance? Separate contexts? Separate fine‑tuned LoRAs?
2. What agent communication protocol within a layer? Direct message passing via shared memory? Broadcast to hypervisor?
3. How to make agents “small” (2–4 bit, <1GB) so they can run on consumer GPUs alongside the main LLM?

### Hypervisor Design
1. Should the hypervisor be a separate fine‑tuned model or a prompt‑engineered instance of the same LLM?
2. How does hypervisor avoid feedback loops / self‑amplification? (clamping, 0.02 gap as a stabilizer)
3. How to implement a “gradient” back‑propagation across layers? (literal autograd on agent outputs?)

### Performance & Scalability
1. Can WebGPU handle concurrent inference for 3+ agents + hypervisor on a single GPU? (time‑slicing, priority queuing)
2. What is the memory footprint of 3 smolagents + a 7B WebGPU model? (estimate)
3. How to horizontally scale a layer across multiple browser tabs or multiple devices (WebRTC cluster)?

### Security & Privacy
1. Since everything is local, how to prevent malicious agents from exfiltrating data? (no network requests, but agents can still log to console – need sandbox)
2. How to verify that a hidden layer is truly hidden from the user? (browser dev tools can inspect memory – maybe use WebAssembly with encrypted memory?)
