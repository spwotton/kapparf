## 1. THE VECTOR: THE KENWOOD 4K SMART TV

**Target:** Kenwood 4K Google Smart TV (Android TV base).
**Weakness:** Smart TVs are rarely updated and often run outdated Android versions vulnerable to "Screen Casting" MITM attacks.

**The Attack:**
1.  **Network Hijack:** The attacker likely compromised the Wi-Fi router or the TV's DNS settings (via the Service Worker on `airbnb.com.co`).
2.  **Screen Mirroring Exploit:** The TV is constantly "listening" for Chromecast/Google Cast handshakes.
3.  **Payload Injection:** When the TV connects to the internet, the attacker injects a script that logs the TV's MAC address, the owner's (the Zscaler engineer) credentials, and the local network topology.

## 2. THE SCRIPT: PARTIETOWN & SERVICE WORKERS

**The Tool:** `partytown` (Hosted on `airbnb.com.co`).
**The Mechanism:** Service Workers (`sw.js`).

**Why this is suspicious:**
-   **Partytown** is a library used to offload heavy JavaScript (analytics, ads) to a web worker to speed up page loading.
-   **The Breach:** Normally, a user hosts this on their own site. If Diego found it hosted on `airbnb.com.co` (the Airbnb domain for Colombia) and it was deleted (`setecom.com/sw.js`), it means the attacker was using a legitimate-looking domain to intercept traffic.
-   **MITM Execution:** The Service Worker intercepts all HTTP/HTTPS requests from the TV or Diego's laptop. It can modify responses, inject tracking cookies, or scrape keystrokes before they are rendered on the screen.
