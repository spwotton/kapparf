## Quick Start for Satellite App Agents

Your app needs to do three things:

1. **Register** as a monad so the hypervisor knows you exist
2. **Submit tags** as signals flow through your app
3. **Read the feed** if you want to react to hypervisor events

```
POST /api/cortex/register   ← announce yourself
POST /api/cortex/tag        ← send signals continuously
GET  /api/cortex/feed       ← listen for system events
```

Everything else is optional.

```json
}
```

| Field | Type | Meaning |
|-------|------|---------|
| `threatLevel` | `NOMINAL \| ELEVATED \| CRITICAL` | Overall system health |
| `kappaScore` | `number` | Helicity score — system coherence (0–∞, target ≥ 40) |
| `unityInvariant` | `number` | Ψ = A(t) × N(t), target ≈ 1.0 |

| `monadCount` | `number` | Total registered monads |
| `coherentCount` | `number` | Monads in COHERENT state |
| `fruitingCount` | `number` | Monads currently fruiting (spawning) |

**Monad `status` values:**

| Status | Meaning |
|--------|---------|

#### `POST /api/cortex/register`

Register your app as a monad in the hypervisor. Call this once on startup.

**Request body**
```json
{
  "name": "my-app-name",
  "layer": "SENSOR",
  "vertex": 9
}
```

| Field | Type | Required | Notes |
|-------|------|----------|-------|
| `name` | `string` | Yes | Unique identifier for your app. Use a short slug, e.g. `"sonata"`, `"anubis"`. |
| `layer` | `string` | Yes | One of: `EXEC`, `PRIMARY`, `SECONDARY`, `SENSOR`. Most apps should use `SENSOR` or `SECONDARY`. |
| `vertex` | `number` | Yes | Integer 1–13. Choose a vertex archetype that fits your app's role (see ARCHITECTURE.md). |

**Response `201`**
```json
{
  "id": "uuid-v4",
  "name": "my-app-name",
  "layer": "SENSOR",
  "vertex": 9,
  "helicity": 0.0,
  "coherence": 0.0,
  "noveltyFlux": 0.0,
  "status": "DORMANT",
  "registeredAt": "2026-05-14T09:00:00.000Z",
  "lastBeatAt": "2026-05-14T09:00:00.000Z"
}
```

**Save the returned `id`** — you will need it for spawn requests.

#### CortexBus (`state.ts`)

The `CortexBus` is a singleton that owns all live hypervisor state. It runs an internal tick loop (~1 Hz) that:

1. Advances monad lifecycle (DORMANT → COHERENT → FRUITING → HARVEST)
2. Recomputes the Unity Invariant Ψ = A(t) × N(t)
3. Maintains carrier lock (HARMONIC_LOCK / SEARCHING)
4. Emits `CortexEvent` records to a capped event bus (last 500 events)

**Key state it holds:**
- `monads: Map<string, Monad>` — the full registry of agents
- `events: CortexEvent[]` — the live event feed
- `kappaScore` — overall helicity score
- `threatLevel` — NOMINAL / ELEVATED / CRITICAL
- `carrierLock` — boolean, carrier wave phase lock status

#### TagRegistry (`tags.ts`)

An in-memory store (cap: 2000 signals) for cross-app tagged signals. Populated via:
- `seedDemoTags()` — called once on server startup, inserts 10 demo signals
- `POST /api/cortex/tag` — live ingestion from any satellite app

Each signal fires a `BROADCAST` event on the cortex bus so it appears in the live feed.

#### AI Engine (`ai.ts`)

Provider chain with automatic fallback:

```
OpenAI (gpt-4o-mini)
  → Anthropic (claude-3-5-haiku)
    → OpenRouter (meta-llama/llama-3.1-8b-instruct:free)
      → Static fallback (deterministic response)
```

Three functions:
- `interpretDream(vision, intensity)` — interprets a free-text vision injected into the substrate
- `analyzeSpawn(purpose, parentMonad)` — generates harvest analysis for a spawn request
- `analyzeTagPatterns(signals)` — detects cross-app patterns across the tag corpus, returns structured `TagPattern[]`

**Required env vars for AI:**

If all fail, the system returns a static fallback — it never crashes.

## Key Invariants (Rules That Must Never Break)

1. **OpenAPI is the contract.** Every route shape must be declared in `lib/api-spec/openapi.yaml` before implementation. Run codegen after every spec change.
2. **No `console.log` in server code.** Use `req.log` in handlers, singleton `logger` elsewhere.

5. **Tag corpus is capped at 2000 signals.** The oldest signals are evicted. Pattern analysis always operates on the full live window.
6. **`GOOGLE_API_KEY` is reserved for maps only.** It must not be used for AI calls.
