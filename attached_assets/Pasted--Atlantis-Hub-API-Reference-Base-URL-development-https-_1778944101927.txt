# Atlantis Hub — API Reference

**Base URL (development):** `https://<your-replit-dev-domain>/api`  
**Base URL (production):** `https://<your-replit-app-domain>/api`  
**Protocol:** HTTP/1.1, JSON bodies  
**Auth:** None currently (internal network assumed)

This is the handoff document for satellite app agents. If you are building an app that connects to the Ω-GOS Living Hypervisor, start here.

---

## Quick Start for Satellite App Agents

Your app needs to do three things:

1. **Register** as a monad so the hypervisor knows you exist
2. **Submit tags** as signals flow through your app
3. **Read the feed** if you want to react to hypervisor events

```
POST /api/cortex/register   ← announce yourself
POST /api/cortex/tag        ← send signals continuously
GET  /api/cortex/feed       ← listen for system events
```

Everything else is optional.

#### `POST /api/cortex/register`

Register your app as a monad in the hypervisor. Call this once on startup.

**Request body**
```json
{
  "name": "my-app-name",
  "layer": "SENSOR",
  "vertex": 9
}
```

| Field | Type | Required | Notes |
|-------|------|----------|-------|
| `name` | `string` | Yes | Unique identifier for your app. Use a short slug, e.g. `"sonata"`, `"anubis"`. |
| `layer` | `string` | Yes | One of: `EXEC`, `PRIMARY`, `SECONDARY`, `SENSOR`. Most apps should use `SENSOR` or `SECONDARY`. |
| `vertex` | `number` | Yes | Integer 1–13. Choose a vertex archetype that fits your app's role (see ARCHITECTURE.md). |

**Response `201`**
```json
{
  "id": "uuid-v4",
  "name": "my-app-name",
  "layer": "SENSOR",
  "vertex": 9,
  "helicity": 0.0,
  "coherence": 0.0,
  "noveltyFlux": 0.0,
  "status": "DORMANT",
  "registeredAt": "2026-05-14T09:00:00.000Z",
  "lastBeatAt": "2026-05-14T09:00:00.000Z"
}
```

**Save the returned `id`** — you will need it for spawn requests.

### Cortex Bus (Event Feed)

#### `GET /api/cortex/feed`

The live cortex bus — all system events from the last 500 entries.

**Query params**

| Param | Type | Default | Notes |
|-------|------|---------|-------|
| `limit` | `number` | `50` | Max events to return |
| `since` | `string` | — | ISO timestamp — only return events after this time |

**Response `200`** — array of `CortexEvent`
```json
[
  {
    "id": "uuid-v4",
    "type": "BROADCAST",
    "source": "sonata",
    "message": "[sonata] signal.resonance_hz → 8.392",
    "timestamp": "2026-05-14T09:03:22.000Z",
    "data": {}
  }
]
```

**Event `type` values:**

| Type | Trigger |
|------|---------|
| `BROADCAST` | Tag submitted, monad registered, or manual broadcast |
| `SPAWN` | Fruiting body catalysed |
| `HARVEST` | Fruiting cycle completed, AI analysis attached |
| `DREAM` | Vision injected into the substrate |
| `TICK` | Internal heartbeat (every ~5s) |
| `HARMONIC_LOCK` | Carrier wave achieved phase lock |
| `PHOENIX` | System reset / rebirth cycle |

**Polling recommendation:** poll every 2–5 seconds. WebSocket is future work.

### Universal Tagging

This is the primary integration surface. Satellite apps emit signals here; the hypervisor builds a corpus and runs AI pattern analysis to detect cross-app emergence.

#### `POST /api/cortex/tag`

Submit a tagged signal from your app.

**Request body**
```json
{
  "sourceApp": "my-app-name",
  "tagKey": "geo.hotspot",
  "tagValue": "cluster_detected",
  "geo": {
    "lat": 51.5,
    "lng": -0.09
  },
  "confidence": 0.92,
  "metadata": {
    "clusterId": "c_77",
    "userCount": 450
  }
}
```

| Field | Type | Required | Notes |
|-------|------|----------|-------|
| `sourceApp` | `string` | Yes | Your app's name/slug. Matches what you used in `register`. |
| `tagKey` | `string` | Yes | Dot-namespaced key. See namespaces below. |
| `tagValue` | `string \| number` | Yes | The signal value. |
| `geo` | `{ lat, lng }` | No | Include if signal is geospatial. |
| `confidence` | `number` | No | 0.0–1.0. Defaults to 1.0. |
| `metadata` | `object` | No | Any additional key/value pairs. |

**Tag key namespaces:**

| Namespace | Use for |
|-----------|---------|
| `geo.*` | Geographic signals: `geo.hotspot`, `geo.density_index`, `geo.boundary_crossing` |
| `climate.*` | Environmental: `climate.co2_ppm`, `climate.temperature_c` |
| `bio.*` | Biometric/biological: `bio.growth_rate`, `bio.photon_flux` |
| `pattern.*` | Detected patterns: `pattern.recurrence`, `pattern.anomaly` |
| `signal.*` | Raw signal readings: `signal.resonance_hz`, `signal.amplitude` |
| `network.*` | Network/infrastructure: `network.latency_ms`, `network.packet_loss` |
| `user.*` | User behaviour: `user.session_duration`, `user.conversion` |
| `app.*` | App-level metrics: `app.error_rate`, `app.throughput` |

**Response `201`**
```json
{
  "id": "uuid-v4",
  "sourceApp": "my-app-name",
  "tagKey": "geo.hotspot",
  "tagValue": "cluster_detected",
  "geo": { "lat": 51.5, "lng": -0.09 },
  "confidence": 0.92,
  "metadata": {},
  "timestamp": "2026-05-14T09:03:22.000Z"
}
```

**Emit tags for any meaningful event in your app.** The AI pattern analysis is only as good as the signal volume across apps. More signals = richer pattern detection.

### AI Pattern Analysis

#### `GET /api/cortex/patterns`

List AI-detected cross-app patterns from the most recent analysis run.

**Response `200`** — array of `TagPattern`
```json
[
  {
    "id": "uuid-v4",
    "name": "Phi Resonance Convergence",
    "description": "Multiple apps are detecting synchronized resonance peaks at phi-ratio intervals.",
    "confidence": 0.87,
    "involvedApps": ["sonata", "anubis"],
    "tagKeys": ["signal.resonance_hz", "pattern.recurrence"],
    "seedIdea": "Build a cross-app harmonic scheduler that coordinates task timing to phi-ratio intervals, reducing destructive interference across services.",
    "centroid": { "lat": 51.5, "lng": -0.1 },
    "detectedAt": "2026-05-14T09:10:00.000Z"
  }
]
```

| Field | Type | Meaning |
|-------|------|---------|
| `name` | `string` | AI-generated pattern name |
| `description` | `string` | What the pattern means |
| `confidence` | `number` | 0–1 confidence score |
| `involvedApps` | `string[]` | Which apps contributed signals |
| `tagKeys` | `string[]` | Which tag keys were involved |
| `seedIdea` | `string` | AI-suggested software system to build from this pattern |
| `centroid` | `GeoPoint \| null` | Geographic centre if pattern has geo dimension |

---

#### `POST /api/cortex/patterns/analyze`

Trigger a fresh AI analysis of the entire tag corpus. This is expensive — call it manually or on a schedule, not on every tag submission.

**Request body:** empty (`{}`)

**Response `200`**
```json
{
  "patterns": [ /* array of TagPattern */ ],
  "analyzedSignals": 142,
  "provider": "openai"
}
```

The `provider` field tells you which AI provider actually ran the analysis (openai / anthropic / openrouter / static-fallback).

This call also fires a `HARVEST` event on the cortex bus.

## Integration Checklist for Satellite App Agents

Use this checklist when connecting a new app to the hypervisor:

```
□ 1. Call POST /api/cortex/register on app startup
      Save the returned monad ID.

□ 2. Identify which tag namespaces apply to your app
      geo.*, user.*, app.*, signal.*, etc.

□ 3. Add POST /api/cortex/tag calls at meaningful events:
      - User completes key action → user.conversion
      - Error spike detected → app.error_rate
      - Geographic cluster found → geo.hotspot
      - Any metric you track → signal.* or app.*

□ 4. Optionally poll GET /api/cortex/feed to react to
      system events (HARVEST, PHOENIX, HARMONIC_LOCK)

□ 5. Optionally expose your own API reference so the
      hypervisor can call back into you (webhook pattern)

□ 6. Report your API base URL back to the Atlantis Hub
      agent so it can be registered in the topology.
```

## Error Responses

All errors follow this shape:

```json
{
  "error": "Human-readable error message"
}
```

| Code | Meaning |
|------|---------|
| `400` | Invalid request body — check your Zod validation errors |
| `404` | Resource not found |
| `409` | Conflict — e.g. spawn rejected because kappaScore < 100 |
| `500` | Internal error — AI provider chain exhausted all fallbacks |

## Notes for Agents Writing Satellite Apps

- **Namespace your tag keys properly.** The AI pattern analysis groups by namespace. Inconsistent naming degrades pattern quality.
- **Always include `confidence`.** Even a rough estimate (0.7 = fairly confident, 0.95 = highly confident) improves pattern weighting.
- **Include `geo` when you have it.** The centroid calculation adds a geographic dimension to patterns — very powerful for location-aware apps.
- **Do not poll `/api/cortex/patterns/analyze`.** It's a heavy AI call. Trigger it from the Atlantis Hub UI or a scheduled cron.
- **Your monad's `status` will advance automatically** as the cortex tick loop evaluates coherence. You do not need to manually update it.
- **Return your API reference** to the Atlantis Hub agent (me) so the hub can register your endpoints and enable bidirectional coordination.
