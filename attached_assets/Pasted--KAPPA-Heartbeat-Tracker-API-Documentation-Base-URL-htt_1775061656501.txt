## Devices

### `POST /devices/register`

Register a new device or update an existing one. Sets device status to online.

**Request Body:**
```json
{
  "deviceId": "echo-pc",          // required, unique identifier
  "name": "Echo Desktop",         // required, display name
  "type": "pc",                   // required, one of: "pc", "phone", "sensor", "iot"
  "os": "windows",                // required, e.g. "windows", "linux", "android", "macos"
  "capabilities": ["cpu", "memory", "disk"],  // optional, string array
  "metadata": {                   // optional, arbitrary JSON
    "hostname": "ECHO-PC",
    "cpu_count": 16,
    "processor": "Intel i7-13700K",
    "ram_total_gb": 32
  }
}
```

**Response (200):**
```json
{
  "id": "uuid-string",
  "deviceId": "echo-pc",
  "name": "Echo Desktop",
  "type": "pc",
  "os": "windows",
  "online": true,
  "ip": null,
  "capabilities": ["cpu", "memory", "disk"],
  "registeredAt": "2026-04-01T16:00:00.000Z",
  "lastHeartbeat": "2026-04-01T16:00:00.000Z",
  "lastDisconnect": null
}
```

---

### `GET /devices`

List all registered devices with optional filtering.

**Query Parameters:**
| Param    | Type    | Description                        |
|----------|---------|------------------------------------|
| `type`   | string  | Filter by device type (pc, phone, sensor, iot) |
| `online` | boolean | Filter by online status            |

**Example:** `GET /devices?type=pc&online=true`

**Response (200):** Array of `DeviceInfo` objects (same shape as register response).

## Heartbeat

### `POST /heartbeat`

Send a heartbeat from a device. Updates the device's online status and last heartbeat timestamp.

**Request Body:**
```json
{
  "deviceId": "echo-pc",       // required
  "metadata": {                 // optional, arbitrary system metrics
    "cpu": 45.2,
    "memory": 62.1,
    "disk": 78.5,
    "uptime": 86400
  }
}
```

**Response (200):**
```json
{
  "ack": true,
  "serverTime": 1775060400000,     // Unix timestamp ms
  "nextExpectedMs": 15000          // Expected interval until next heartbeat
}
```

---

### `GET /heartbeat/:deviceId/history`

Get heartbeat history for a device.

**Path Parameters:**
| Param      | Type   | Description        |
|------------|--------|--------------------|
| `deviceId` | string | The device's unique ID |

**Query Parameters:**
| Param   | Type    | Default | Description                    |
|---------|---------|---------|--------------------------------|
| `hours` | integer | 24      | How many hours of history      |
| `limit` | integer | 1000    | Max number of records returned |

**Response (200):** Array of `HeartbeatRecord` objects:
```json
[
  {
    "id": "uuid",
    "deviceId": "echo-pc",
    "timestamp": "2026-04-01T16:15:00.000Z",
    "latencyMs": null,
    "metadata": { "cpu": 45.2, "memory": 62.1 }
  }
]
```

## Sensors

### `POST /sensors`

Ingest multiple sensor readings for a device.

**Request Body:**
```json
{
  "deviceId": "rpi-sensor-01",    // required
  "readings": [                    // required, array of readings
    {
      "sensorType": "temperature", // required
      "timestamp": 1775060400000,  // required, Unix timestamp ms
      "values": {                  // required, arbitrary sensor data
        "celsius": 42.5,
        "fahrenheit": 108.5,
        "location": "CPU"
      }
    },
    {
      "sensorType": "humidity",
      "timestamp": 1775060400000,
      "values": { "percent": 55.2 }
    }
  ]
}
```

**Response (200):**
```json
{ "ingested": 2 }
```

---

**Query Parameters:**
| Param   | Type    | Default | Description                   |

## KAPPA System

### `GET /kappa/status`

Full system overview. Returns all devices, alerts, latest heartbeat info, and server uptime.

**Response (200):**
```json
{
  "devices": [ /* array of DeviceInfo */ ],
  "onlineCount": 2,
  "alerts": [ /* array of active Alert */ ],
  "recentHeartbeats": {
    "echo-pc": {
      "lastSeen": "2026-04-01T16:15:00.000Z",
      "latencyMs": null,
      "metadata": { "cpu": 45.2, "memory": 62.1 }
    }
  },
  "serverUptime": 3600.5
}
```

---

### `POST /kappa/command`

Send a command to a device through its active WebSocket connection. The device must be online and connected via WebSocket.

**Request Body:**
```json
{
  "deviceId": "echo-pc",          // required
  "command": "collect_sensors",    // required
  "args": {}                       // optional, arbitrary arguments
}
```

**Available commands:** `collect_sensors`, `run_scan`, `screenshot`, `process_list`, `network_dump` (or any custom string — the agent decides how to handle it).

**Response (200):**
```json
{ "sent": true }    // or { "sent": false } if device is not connected
```

## Agent Scripts

Downloadable agent scripts that devices run to connect to the KAPPA server.

### `GET /agents/pc`
Returns a Python agent script for desktop (Windows/Linux/Mac). Uses `psutil` and `websocket-client`. Content-Type: `text/plain`.

### `GET /agents/phone`
Returns a Python agent script for Termux (Android). Uses `termux-api` commands for battery, wifi, location data. Content-Type: `text/plain`.

### `GET /agents/bash`
Returns a minimal bash script using `curl` to register and send heartbeats via the REST API. Content-Type: `text/plain`.

## WebSocket Protocol

**Endpoint:** `wss://<your-domain>/ws?deviceId=<device-id>`

The WebSocket connection provides real-time bidirectional communication between agents and the server.

### Client → Server Messages

#### Register
```json
{
  "type": "register",
  "name": "My Device",
  "deviceType": "pc",
  "os": "windows",
  "capabilities": ["cpu", "memory", "disk"],
  "metadata": { "hostname": "ECHO-PC" }
}
```
**Server reply:**
```json
{
  "type": "registered",
  "deviceId": "echo-pc",
  "serverTime": 1775060400000,
  "heartbeatIntervalMs": 15000
}
```

#### Heartbeat
```json
{
  "type": "heartbeat",
  "cpu": 45.2,
  "memory": 62.1,
  "disk": 78.5
}
```
**Server reply:**
```json
{
  "type": "heartbeat_ack",
  "serverTime": 1775060400000
}
```

#### Sensor Data
```json
{
  "type": "sensors",
  "readings": [
    {
      "sensorType": "temperature",
      "timestamp": 1775060400000,
      "values": { "celsius": 42.5 }
    }
  ]
}
```

#### Alert
```json
{
  "type": "alert",
  "message": "High CPU usage detected",
  "severity": 2,
  "metadata": { "cpu_percent": 98.5 }
}
```

### Server → Client Messages

#### Command (sent via `POST /kappa/command`)
```json
{
  "type": "command",
  "command": "collect_sensors",
  "args": {}
}
```

### Connection Lifecycle
1. Client connects to `wss://<domain>/ws?deviceId=my-device`
2. Client sends a `register` message
3. Server replies with `registered` including the `heartbeatIntervalMs`
4. Client sends `heartbeat` messages at the specified interval
5. On disconnect, server marks device as offline and records `lastDisconnect`
6. If a device was previously offline, a `reconnect` alert (severity 1) is created on re-registration

## Database Schema

### `devices`
| Column           | Type                     | Notes                           |
|------------------|--------------------------|---------------------------------|
| `id`             | varchar (UUID)           | Primary key, auto-generated     |
| `device_id`      | text                     | Unique, human-readable ID       |
| `name`           | text                     | Display name                    |
| `type`           | text                     | pc, phone, sensor, iot          |
| `os`             | text                     | Operating system                |
| `ip`             | text (nullable)          | Last known IP                   |
| `capabilities`   | text[]                   | Array of capability strings     |
| `metadata`       | jsonb                    | Arbitrary device metadata       |
| `online`         | boolean                  | Current online status           |
| `registered_at`  | timestamptz              | Registration timestamp          |
| `last_heartbeat` | timestamptz (nullable)   | Last heartbeat received         |
| `last_disconnect`| timestamptz (nullable)   | Last time device went offline   |

### `heartbeats`
| Column       | Type                   | Notes                          |
|--------------|------------------------|--------------------------------|
| `id`         | varchar (UUID)         | Primary key                    |
| `device_id`  | text                   | FK → devices.device_id (CASCADE) |
| `timestamp`  | timestamptz            | When heartbeat was received    |
| `latency_ms` | real (nullable)        | Round-trip latency             |
| `metadata`   | jsonb                  | System metrics (cpu, memory, etc.) |

### `alerts`
| Column         | Type                   | Notes                         |
|----------------|------------------------|-------------------------------|
| `id`           | varchar (UUID)         | Primary key                   |
| `device_id`    | text                   | FK → devices.device_id (CASCADE) |
| `type`         | text                   | timeout, reconnect, manual, anomaly |
| `message`      | text                   | Human-readable alert message  |
| `severity`     | integer                | 1=info, 2=warning, 3=critical |
| `metadata`     | jsonb                  | Extra alert context           |
| `created_at`   | timestamptz            | When alert was created        |
| `acknowledged` | boolean                | Whether alert has been dismissed |

### `sensor_readings`
| Column        | Type                   | Notes                         |
|---------------|------------------------|-------------------------------|
| `id`          | varchar (UUID)         | Primary key                   |
| `device_id`   | text                   | FK → devices.device_id (CASCADE) |
| `sensor_type` | text                   | e.g. temperature, humidity, battery |
| `timestamp`   | timestamptz            | When reading was taken        |
| `values`      | jsonb                  | Sensor data payload           |

## Background Jobs

### Heartbeat Monitor (every 15 seconds)
- Checks all devices marked `online`
- If `last_heartbeat` is older than 45 seconds (configurable via `HEARTBEAT_TIMEOUT_MS` env var), marks device as offline
- Creates a severity 2 (warning) `timeout` alert for the device

### Data Cleanup (every 1 hour)
- Deletes heartbeat records older than 24 hours
- Deletes sensor readings older than 6 hours
- Deletes acknowledged alerts older than 7 days

## Quick Start (Connecting a Device)

### Via REST API (simplest)
```bash
# Register
curl -X POST https://<domain>/api/devices/register \
  -H "Content-Type: application/json" \
  -d '{"deviceId":"my-pc","name":"My PC","type":"pc","os":"linux"}'

# Send heartbeat
curl -X POST https://<domain>/api/heartbeat \
  -H "Content-Type: application/json" \
  -d '{"deviceId":"my-pc","metadata":{"cpu":25.0,"memory":40.0}}'
```

### Via WebSocket (recommended for persistent connections)
```javascript
const ws = new WebSocket("wss://<domain>/ws?deviceId=my-pc");

ws.onopen = () => {
  ws.send(JSON.stringify({
    type: "register",
    name: "My PC",
    deviceType: "pc",
    os: "linux",
    capabilities: ["cpu", "memory"]
  }));

  setInterval(() => {
    ws.send(JSON.stringify({
      type: "heartbeat",
      cpu: Math.random() * 100,
      memory: Math.random() * 100
    }));
  }, 15000);
};
```

### Via Downloadable Agent Scripts
- **PC (Python):** `GET /api/agents/pc` — full-featured agent using psutil
- **Phone (Termux):** `GET /api/agents/phone` — Android agent using termux-api
- **Bash (Minimal):** `GET /api/agents/bash` — lightweight curl-based heartbeat
