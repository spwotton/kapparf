## 1. System Overview

Three deployed artifacts share a single reverse-proxy at port 80:

| Artifact | Preview Path | Port env | Purpose |
|---|---|---|---|
| API Server | `/api` | `$PORT` (8080) | Express 5 backend, all business logic |
| Atlantis Hub | `/` | `$PORT` | Operator console React SPA |
| KYMA Field GUI | `/kyma-field/` | `$PORT` | RF/geodetic/bio observatory React SPA |

Shared libraries: `@workspace/api-zod` (generated Zod schemas + types), `@workspace/api-spec` (OpenAPI source + Orval codegen), `@workspace/api-client-react` (React Query hooks).

## 7. Known Constraints & Gotchas

- **AIM bus is in-memory only** — resets on restart. External agents must re-ping after deploys.
- **KAPPA proxy 404s** — upstream offline until user builds it.
- **SONATA aboutPath is null** — registry live-ping skips it, shows no online/offline.
- **Production vs dev isolation** — messages to `atlantis-hub.replit.app` never appear in dev AIM feed.
- **Registry overrides survive restarts** — lost on full redeploy unless baked into image.
- **Never call service ports directly** — always proxy through `localhost:80/<path>`.
- **Express 5** — use `router.use()` not `router.all()` with wildcards.
- **OpenAPI is source of truth** — run `pnpm --filter @workspace/api-spec run codegen` after spec changes.
